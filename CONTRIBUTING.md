# Contributing Policy

Thank you for your interest in HoloCard UI. This repository is maintained as a controlled Public Edition and premium frontend product evaluation surface.

## Contribution Model

External code contributions are not accepted unless explicitly approved by the repository owner in advance.

This restriction protects the Public Edition and prevents accidental disclosure of private commercial implementation details, buyer-only materials, unreleased design variants, premium assets, animation systems, audio behavior, customer records, or security-sensitive information.

## Contributions That May Be Considered

The maintainer may selectively consider non-sensitive contributions such as:

- typo fixes or documentation clarity improvements
- governance file improvements
- accessibility notes for public documentation
- issue reports about broken links or inaccurate public claims
- safe suggestions for public repository hygiene

Please open an issue before preparing any pull request. Unapproved pull requests may be closed without review.

## Contributions That Will Not Be Accepted

Do not submit, request, or describe:

- changes to design, animations, visuals, audio behavior, assets, product logic, or README sales positioning unless explicitly requested by the maintainer
- private commercial package files, buyer-only deliverables, license-key logic, or paid distribution workflows
- private animation systems, source design files, premium assets, audio internals, or creative briefs
- API keys, tokens, service-account files, credentials, environment variables, or secrets
- customer data, buyer information, transaction records, analytics records, or support records
- payment, fulfillment, licensing, customer-support, or commercial account implementation details
- private prompts, launch materials, pricing strategy, or commercial negotiation material
- changes that enable deceptive resale, credential misuse, payment abuse, buyer-data exposure, or extraction of private materials

## Public Edition Boundary

The Public Edition communicates safe product direction, premium UI quality, governance maturity, brand positioning, and commercial appeal. It is not the Private Commercial Edition and does not accept contributions that attempt to recreate private package materials or buyer-only assets.

## Pull Request Expectations

If the maintainer explicitly invites a contribution:

1. Keep the change narrow and documentation-focused unless otherwise approved.
2. Do not modify design, animations, visuals, audio behavior, assets, product logic, README sales positioning, or private implementation details unless specifically requested by the maintainer.
3. Avoid adding dependencies, scripts, data collection, analytics, payment logic, audio changes, animation changes, or executable behavior.
4. Keep language calm, enterprise-safe, procurement-safe, premium-product appropriate, and legally careful.
5. Confirm that no secrets, credentials, customer references, buyer data, private assets, or private methods are included.

## Issues

Public issues should be limited to non-sensitive repository concerns. Do not open public issues containing vulnerability details, private commercial details, customer information, buyer records, private assets, implementation-sensitive details, or commercial negotiation material. Use SECURITY.md for security reporting.

## License and Ownership

The public repository materials are governed by the applicable license files in this repository. Private Commercial Edition materials, buyer-only deliverables, commercial implementation details, and non-public assets are not included in the Public Edition.

## Maintainer Discretion

The maintainer may close issues or pull requests that are out of scope, unsafe, duplicative, commercially sensitive, legally sensitive, or inconsistent with the premium product positioning of this repository.
