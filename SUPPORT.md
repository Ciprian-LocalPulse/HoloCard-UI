# Support

This repository is the Public Edition of HoloCard UI. Support is limited to public documentation, governance, repository hygiene, and safe evaluation questions.

## Public Support

For non-sensitive public questions, you may use GitHub Issues for:

- broken links
- documentation errors
- unclear public wording
- governance file corrections
- repository metadata or public-readiness feedback

Please do not include security-sensitive information, private implementation details, buyer information, transaction records, license keys, private assets, screenshots of private accounts, or commercial fulfillment assumptions in public issues.

## Commercial Inquiries

For licensing, commercial use, customization discussions, design-partner conversations, acquisition inquiries, or Private Commercial Edition access, contact the repository owner through the public GitHub profile or the contact path referenced in the repository README.

Commercial discussions should not be opened as public issues when they involve pricing, private capabilities, buyer requirements, paid package delivery, customer information, licensing details, or confidential evaluation criteria.

## Security Reporting

Security issues must follow SECURITY.md.

Do not publish vulnerability details, secrets, tokens, buyer data, payment details, private asset references, license-key details, commercial account information, or implementation-sensitive findings in public issues or discussions.

## Public Edition vs. Private Commercial Edition

Public support covers the Public Edition only. It does not include:

- private deployment support
- paid-product fulfillment support through GitHub issues
- custom visual or animation implementation
- audio behavior customization
- private asset transfer
- license-key, payment, or customer account support in public channels
- commercial architecture transfer

Private Commercial Edition support, if available, requires a separate written agreement or authorized commercial purchase.

## Response Expectations

This repository is maintained as a public product and governance surface. Public issue response times are best-effort and depend on severity, relevance, and maintainer availability.

Security reports and commercial inquiries are prioritized through the appropriate private channels.
